NetwContext
===========

**Inherits:** `RefCounted <https://docs.godotengine.org/en/stable/classes/class_refcounted.html>`__

Composite context providing tree-level, service-level, scene-level, and entity-level access via :ref:`tree<class_NetwContext_property_tree>`, :ref:`services<class_NetwContext_property_services>`, :ref:`scene<class_NetwContext_property_scene>`, and :ref:`entity<class_NetwContext_property_entity>`.

.. rst-class:: classref-introduction-group

Description
-----------

Obtain via :ref:`Netw.ctx()<class_Netw_method_ctx>` or :ref:`for_node()<class_NetwContext_method_for_node>`.

::

    var ctx := Netw.ctx(self)
    ctx.services.get_scene_manager()
    if ctx.has_scene():
        await ctx.scene.wait_for_players(4)
    ctx.entity.collecting_spawn_properties.connect(_on_collect)

\ **Per-member nullability:** Each facade resolves independently under its own constraints, so any of :ref:`tree<class_NetwContext_property_tree>`, :ref:`services<class_NetwContext_property_services>`, :ref:`scene<class_NetwContext_property_scene>`, or :ref:`entity<class_NetwContext_property_entity>` may be ``null``: 

- :ref:`tree<class_NetwContext_property_tree>`, :ref:`services<class_NetwContext_property_services>` need an enclosing :ref:`MultiplayerTree<class_MultiplayerTree>` in the live tree. 

- :ref:`scene<class_NetwContext_property_scene>` needs an enclosing :ref:`MultiplayerScene<class_MultiplayerScene>`. 

- :ref:`entity<class_NetwContext_property_entity>` only needs a parent chain, so it resolves even on orphans (e.g. during `Node.NOTIFICATION_PARENTED <https://docs.godotengine.org/en/stable/classes/class_node.html#class_Node_constant_NOTIFICATION_PARENTED>`__).  **Listen-Server checks:** Access :ref:`tree<class_NetwContext_property_tree>` to check :ref:`NetwTree.is_listen_server()<class_NetwTree_method_is_listen_server>` when writing custom RPCs.
