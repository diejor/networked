SaveComponent
=============

**Inherits:** :ref:`ProxySynchronizer<class_ProxySynchronizer>` **<** `MultiplayerSynchronizer <https://docs.godotengine.org/en/stable/classes/class_multiplayersynchronizer.html>`__

Syncs and persists an entity's scene state to :ref:`database<class_SaveComponent_property_database>`.

.. rst-class:: classref-introduction-group

Description
-----------

Properties picked in the Replication panel are synchronized across the network and flushed on change.  State bounces through :ref:`database<class_SaveComponent_property_database>`: every write is persisted, and :ref:`hydrate()<class_SaveComponent_method_hydrate>` reloads from the database on spawn. 

::

    %SaveComponent.set_value(&"gold", 500)
    var gold := %SaveComponent.get_value(&"gold", 0)

\ See :ref:`hydrate()<class_SaveComponent_method_hydrate>` and :ref:`flush()<class_SaveComponent_method_flush>` for the load/save API.
