SpawnerComponent
================

**Inherits:** `MultiplayerSynchronizer <https://docs.godotengine.org/en/stable/classes/class_multiplayersynchronizer.html>`__

Orchestration point for a networked entity.

.. rst-class:: classref-introduction-group

Description
-----------

`replication_config <https://docs.godotengine.org/en/stable/classes/class_spawnercomponent.html#class_SpawnerComponent_property_replication_config>`__ bundles properties into the spawn packet so initial state arrives with the entity. Sibling components contribute paths through :ref:`NetwEntity.contribute_spawn_property()<class_NetwEntity_method_contribute_spawn_property>` from their own `Node.NOTIFICATION_PARENTED <https://docs.godotengine.org/en/stable/classes/class_node.html#class_Node_constant_NOTIFICATION_PARENTED>`__; the inspector's Replication panel can also pre-populate the list (its flags are coerced to spawn-only at runtime).  



The synchronizer itself always has multiplayer authority ``1`` (server), regardless of the `owner <https://docs.godotengine.org/en/stable/classes/class_spawnercomponent.html#class_SpawnerComponent_property_owner>`__'s authority mode. This guarantees the server can always issue spawn and despawn commands, even for client-authoritative entities whose owner has a peer-specific authority.  



Contributions **must** happen at parented-time, not at tree-entered: Godot reads `replication_config <https://docs.godotengine.org/en/stable/classes/class_spawnercomponent.html#class_SpawnerComponent_property_replication_config>`__ for spawn-decode between PackedScene instantiation and tree entry, so anything added at `NetwEntity.spawning <https://docs.godotengine.org/en/stable/classes/class_netwentity.html#class_NetwEntity_signal_spawning>`__ time is too late for the spawn packet.  



See :ref:`instantiate_from()<class_SpawnerComponent_method_instantiate_from>`, :ref:`spawn_under()<class_SpawnerComponent_method_spawn_under>`, :ref:`spawn_player()<class_SpawnerComponent_method_spawn_player>`, and :ref:`despawn()<class_SpawnerComponent_method_despawn>` for the spawn/despawn API.  Siblings react to the spawn lifecycle via `NetwEntity.spawning <https://docs.godotengine.org/en/stable/classes/class_netwentity.html#class_NetwEntity_signal_spawning>`__ (replaces the older ``_on_entity_spawning`` propagate-call hook).

::

    func _notification(what: int) -> void:
        if what == NOTIFICATION_PARENTED:
            var entity := Netw.ctx(self).entity
            entity.contribute_spawn_property(NodePath("..:my_property"))
            entity.spawning.connect(_on_spawning)

    func _on_spawning() -> void:
        if multiplayer.is_server():
            hydrate_from_db()
