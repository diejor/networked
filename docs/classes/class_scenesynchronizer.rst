SceneSynchronizer
=================

**Inherits:** `MultiplayerSynchronizer <https://docs.godotengine.org/en/stable/classes/class_multiplayersynchronizer.html>`__

Manages per-scene synchronization visibility so each peer only receives data for their scene.

.. rst-class:: classref-introduction-group

Description
-----------

Attach this synchronizer to a ``Scene`` node. Call :ref:`track_node()<class_SceneSynchronizer_method_track_node>` for each entity node to register it; the synchronizer will then restrict replication so only peers inside this scene receive updates. Peer membership is tracked in :ref:`connected_peers<class_SceneSynchronizer_property_connected_peers>`.

|void| **track_node**\ (\ node\: `Node <https://docs.godotengine.org/en/stable/classes/class_node.html>`__\ ) :ref:`🔗<class_SceneSynchronizer_method_track_node>`

Binds a node's lifecycle to the scene's visibility filters.  When ``node`` is already in the tree (e.g., a preplaced entity registering during its own `Node.tree_entered <https://docs.godotengine.org/en/stable/classes/class_node.html#class_Node_signal_tree_entered>`__ handler), the spawn-side bookkeeping fires immediately so visibility filters are applied without waiting for the next tree-entry.
