MultiplayerTree
===============

**Inherits:** `Node <https://docs.godotengine.org/en/stable/classes/class_node.html>`__

Core networking node that bridges a :ref:`BackendPeer<class_BackendPeer>` transport with Godot's ``SceneMultiplayer`` API.

.. rst-class:: classref-introduction-group

Description
-----------

Assign a :ref:`BackendPeer<class_BackendPeer>` (e.g. :ref:`ENetBackend<class_ENetBackend>`, :ref:`WebSocketBackend<class_WebSocketBackend>`), then call :ref:`host()<class_MultiplayerTree_method_host>` or :ref:`join()<class_MultiplayerTree_method_join>` to start a session. Add a :ref:`MultiplayerSceneManager<class_MultiplayerSceneManager>` as a child to manage multiple scenes, or drop a world scene directly as a child to use a single auto-configured scene.

::

    # Server
    await multiplayer_tree.host()

    # Client
    var err = await multiplayer_tree.join("192.168.1.5", "PlayerOne")
    if err != OK:
        push_error("Join failed: %s" % error_string(err))

.. _class_MultiplayerTree_method_adopt_peer:

.. rst-class:: classref-method

Error **adopt_peer**\ (\ peer\: `MultiplayerPeer <https://docs.godotengine.org/en/stable/classes/class_multiplayerpeer.html>`__, join_payload\: :ref:`JoinPayload<class_JoinPayload>` = null\ ) :ref:`🔗<class_MultiplayerTree_method_adopt_peer>`

Adopts a pre-connected ``peer`` without going through a :ref:`BackendPeer<class_BackendPeer>`.  For transports (e.g. Steam lobbies) where the peer is produced by an external lobby flow rather than by :ref:`host()<class_MultiplayerTree_method_host>` / :ref:`join()<class_MultiplayerTree_method_join>`. The peer must be connecting or connected; client adoption waits for Godot to emit `connected_to_server <https://docs.godotengine.org/en/stable/classes/class_multiplayertree.html#class_MultiplayerTree_signal_connected_to_server>`__ before finalizing the session. 



If ``join_payload`` is provided, it is automatically submitted to the server via :ref:`submit_join()<class_MultiplayerTree_method_submit_join>` once adoption is complete. 



Derives :ref:`role<class_MultiplayerTree_property_role>` from ``peer.get_unique_id()``: ``1`` -> ``LISTEN_SERVER``, otherwise ``CLIENT``. 



Returns ``ERR_INVALID_PARAMETER`` if ``peer`` is ``null`` or already disconnected. Asserts the tree is ``OFFLINE``.
