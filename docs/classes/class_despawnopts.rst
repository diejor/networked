DespawnOpts
===========

**Inherits:** `RefCounted <https://docs.godotengine.org/en/stable/classes/class_refcounted.html>`__

Options bag for :ref:`SpawnerComponent.despawn()<class_SpawnerComponent_method_despawn>`.

.. rst-class:: classref-introduction-group

Description
-----------

Carries the knobs that control teardown behavior. Built as a ``RefCounted`` so future options (e.g., delayed-free for death animations) can be added without breaking call sites.

`StringName <https://docs.godotengine.org/en/stable/classes/class_stringname.html>`__ **reason** :ref:`🔗<class_DespawnOpts_property_reason>`

Recorded on the despawn span and forwarded to the `SpawnerComponent.despawning <https://docs.godotengine.org/en/stable/classes/class_spawnercomponent.html#class_SpawnerComponent_signal_despawning>`__ signal so user code can branch on the cause. Common values: ``&"peer_disconnected"``, ``&"killed"``, ``&"collected"``, ``&"timeout"``.

.. _class_DespawnOpts_property_flush_save:

.. rst-class:: classref-property

`bool <https://docs.godotengine.org/en/stable/classes/class_bool.html>`__ **flush_save** = ``true`` :ref:`🔗<class_DespawnOpts_property_flush_save>`

When ``true`` (default), ``SaveComponent.flush`` is called on the despawning node before authority revert and queue_free. A non-OK return is logged at error level and the despawn proceeds - from the caller's perspective despawn is infallible.

.. _class_DespawnOpts_property_defer_free:

.. rst-class:: classref-property

`bool <https://docs.godotengine.org/en/stable/classes/class_bool.html>`__ **defer_free** = ``true`` :ref:`🔗<class_DespawnOpts_property_defer_free>`

When ``true`` (default), the `Node.queue_free() <https://docs.godotengine.org/en/stable/classes/class_node.html#class_Node_method_queue_free>`__ call is deferred. This guarantees the engine's next process step sees the authority change before the node leaves the tree, which fixes the race where a ``MultiplayerSynchronizer`` tries to push state from a freed authority peer.
