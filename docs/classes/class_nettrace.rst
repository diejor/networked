NetTrace
========

**Inherits:** `RefCounted <https://docs.godotengine.org/en/stable/classes/class_refcounted.html>`__

Static access point for the Networked span tracing system.

.. rst-class:: classref-introduction-group

Description
-----------

Opens cross-frame call stacks (:ref:`NetSpan<class_NetSpan>` / :ref:`NetPeerSpan<class_NetPeerSpan>`) and tracks which span is currently active so :ref:`NetworkedDebugReporter<class_NetworkedDebugReporter>` can attach C++ errors to the right operation. 



All state lives in static variables that persist for the process lifetime. :ref:`NetworkedDebugReporter<class_NetworkedDebugReporter>` calls :ref:`reset()<class_NetTrace_method_reset>` at the start of each debug session to clear any stale spans from a previous run. 



\ **Usage:**\ 

::

    var span = Netw.dbg.peer_span(self, "lobby_spawn", peers)
    span.step("spawners_registering")
    # ...operation...
    span.end()       # clean close
    span.fail("err") # failure close

Opens a new general-purpose span and pushes it onto the active stack. 



Returns a no-op :ref:`NetSpan<class_NetSpan>` (empty :ref:`NetSpan.id<class_NetSpan_property_id>`) when the debugger is not active, so consumer code needs no guards. 



Pass ``context`` (typically ``self``) to automatically route the span to the correct :ref:`MultiplayerTree<class_MultiplayerTree>`.
