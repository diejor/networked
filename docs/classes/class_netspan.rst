NetSpan
=======

**Inherits:** `RefCounted <https://docs.godotengine.org/en/stable/classes/class_refcounted.html>`__

Cross-frame call stack entry for the Networked debugger.

.. rst-class:: classref-introduction-group

Description
-----------

Tracks a named operation across multiple process frames, recording a trail of named checkpoints and a final outcome (clean close or failure). 



Do not instantiate directly, use :ref:`NetTrace.begin()<class_NetTrace_method_begin>` or :ref:`NetTrace.begin_peer()<class_NetTrace_method_begin_peer>`.

:ref:`NetSpan<class_NetSpan>` **step_warn**\ (\ step_label\: `String <https://docs.godotengine.org/en/stable/classes/class_string.html>`__, message\: `String <https://docs.godotengine.org/en/stable/classes/class_string.html>`__ = "", data\: `Dictionary <https://docs.godotengine.org/en/stable/classes/class_dictionary.html>`__ = {}\ ) :ref:`🔗<class_NetSpan_method_step_warn>`

Records a non-fatal warning checkpoint within this span without closing it. 



Use this for conditions that are suspicious but not immediately fatal — the span stays OPEN for further steps. For a fatal outcome, use :ref:`fail()<class_NetSpan_method_fail>`. 



Sends ``networked:span_step_warn`` and emits `push_warning() <https://docs.godotengine.org/en/stable/classes/class_netspan.html#class_NetSpan_method_push_warning>`__.

.. _class_NetSpan_method_checkpoint:

.. rst-class:: classref-method

:ref:`CheckpointToken<class_CheckpointToken>` **checkpoint**\ (\ step_label\: `String <https://docs.godotengine.org/en/stable/classes/class_string.html>`__ = ""\ ) :ref:`🔗<class_NetSpan_method_checkpoint>`

Captures the current span state as a :ref:`CheckpointToken<class_CheckpointToken>` for causal linking. 



Pass the returned token to :ref:`NetTrace.begin()<class_NetTrace_method_begin>` or :ref:`NetTrace.begin_peer()<class_NetTrace_method_begin_peer>` via the ``follows_from`` parameter to declare an explicit causal relationship between this span and the new one. 



\ ``step_label`` is optional - use it when the token represents a specific step within this span rather than the span as a whole.
