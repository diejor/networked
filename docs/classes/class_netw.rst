Netw
====

**Inherits:** `Object <https://docs.godotengine.org/en/stable/classes/class_object.html>`__

Entry point for the Networked addon.

.. rst-class:: classref-introduction-group

Description
-----------

Use :ref:`ctx()<class_Netw_method_ctx>` to obtain a :ref:`NetwContext<class_NetwContext>` from any node inside a multiplayer session. The context gives you access to three sub-facades: 

- :ref:`NetwContext.tree<class_NetwContext_property_tree>` -- network operations (pause, kick, disconnect). See :ref:`NetwTree<class_NetwTree>`. 

- :ref:`NetwContext.services<class_NetwContext_property_services>` -- backend systems such as the :ref:`NetworkClock<class_NetworkClock>` and :ref:`MultiplayerSceneManager<class_MultiplayerSceneManager>`, plus custom services you register yourself. See :ref:`NetwServices<class_NetwServices>`. 

- :ref:`NetwContext.scene<class_NetwContext_property_scene>` -- lobby logic (readiness gates, countdowns, player waiting). See :ref:`NetwScene<class_NetwScene>`.  



\ **Other Entry Points**  

- :ref:`dbg<class_Netw_property_dbg>` -- structured logging and causal tracing. See :ref:`NetwDbg<class_NetwDbg>`.  



\ **Security note:** ``any_peer`` RPCs broadcast to every peer when called with ``.rpc()``. Always use ``.rpc_id(1)`` for client-to-server requests and validate the sender inside the handler. 



\ **Note:** Always check :ref:`NetwContext.is_valid()<class_NetwContext_method_is_valid>` before caching a context reference. The underlying :ref:`MultiplayerTree<class_MultiplayerTree>` may be freed during disconnect or scene changes.
