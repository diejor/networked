TubeBackend
===========

**Inherits:** :ref:`BackendPeer<class_BackendPeer>` **<** `Resource <https://docs.godotengine.org/en/stable/classes/class_resource.html>`__

:ref:`BackendPeer<class_BackendPeer>` implementation that delegates transport to a duck-typed TubeClient node.

.. rst-class:: classref-introduction-group

Description
-----------

Assign a ``NodePath`` to a TubeClient node in the scene. Call :ref:`MultiplayerTree.host()<class_MultiplayerTree_method_host>` or :ref:`MultiplayerTree.join()<class_MultiplayerTree_method_join>` as normal - the TubeClient owns its own ``MultiplayerAPI``, so this backend asks the tree to adopt it via `MultiplayerTree._adopt_api() <https://docs.godotengine.org/en/stable/classes/class_multiplayertree.html#class_MultiplayerTree_private_method__adopt_api>`__. 



\ **Service Registration:** During :ref:`setup()<class_TubeBackend_method_setup>`, this backend registers the ``TubeClient`` node as a session-wide service.

::

    # Retrieve the TubeClient from a component
    var tube = ctx.services.get_service(TubeClient)
    if tube:
        print("Session ID: ", tube.session_id)
