LobbyProvider
=============

**Inherits:** `Node <https://docs.godotengine.org/en/stable/classes/class_node.html>`__

**Inherited By:** :ref:`SteamLobbyProvider<class_SteamLobbyProvider>`

Abstract base for lobby providers (Steam, Discord, custom matchmaking, ...).

.. rst-class:: classref-introduction-group

Description
-----------

A **LobbyProvider** runs as a descendant of :ref:`MultiplayerTree<class_MultiplayerTree>` and owns the lifecycle of an external lobby system: creating, browsing, joining, leaving lobbies, and producing a connected ``MultiplayerPeer``. After a peer is ready, callers (typically a lobby UI) hand it to :ref:`NetwTree.adopt_peer()<class_NetwTree_method_adopt_peer>` - or use :ref:`bind()<class_LobbyProvider_method_bind>` to do that plus wire the standard tree-to-lobby sync. 



Implementations should: 

- Register with ``NetwServices.register(self, LobbyProvider)`` on ``_enter_tree``. 

- Emit `peer_ready <https://docs.godotengine.org/en/stable/classes/class_lobbyprovider.html#class_LobbyProvider_signal_peer_ready>`__ once a connected peer exists. 

- Emit `lobby_list_updated <https://docs.godotengine.org/en/stable/classes/class_lobbyprovider.html#class_LobbyProvider_signal_lobby_list_updated>`__ with clear-then-fill semantics. 

- Treat :ref:`create_lobby()<class_LobbyProvider_method_create_lobby>` and :ref:`join_lobby()<class_LobbyProvider_method_join_lobby>` as fire-and-forget; completion is reported via signals.

.. _class_LobbyProvider_method_bind:

.. rst-class:: classref-method

Error **bind**\ (\ tree\: :ref:`NetwTree<class_NetwTree>`, join_payload\: :ref:`JoinPayload<class_JoinPayload>` = null\ ) :ref:`🔗<class_LobbyProvider_method_bind>`

Adopts the produced peer onto ``tree`` and wires standard lifecycle sync (advertised member count, leave-on-disconnect).  If ``join_payload`` is provided, it is passed to :ref:`NetwTree.adopt_peer()<class_NetwTree_method_adopt_peer>` to trigger the session join.  Call once per lobby session after `peer_ready <https://docs.godotengine.org/en/stable/classes/class_lobbyprovider.html#class_LobbyProvider_signal_peer_ready>`__ (or after `lobby_created <https://docs.godotengine.org/en/stable/classes/class_lobbyprovider.html#class_LobbyProvider_signal_lobby_created>`__ / `lobby_joined <https://docs.godotengine.org/en/stable/classes/class_lobbyprovider.html#class_LobbyProvider_signal_lobby_joined>`__, which imply it).
