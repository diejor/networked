TopologyValidator
=================

**Inherits:** `RefCounted <https://docs.godotengine.org/en/stable/classes/class_refcounted.html>`__

Developer-only topology validation for the Networked addon.

.. rst-class:: classref-introduction-group

Description
-----------

Contains static helpers that inspect the synchronizer topology of a player node: expected vs actual counts, cache vs live diff, and virtual property constraint checks. 



\ **Never import this file from production components.** (SaveComponent, SceneSynchronizer, SpawnerComponent, etc.) 



Use from: tests, ``@tool`` scripts, debugger panels.

`int <https://docs.godotengine.org/en/stable/classes/class_int.html>`__ **expected_sync_count**\ (\ node\: `Node <https://docs.godotengine.org/en/stable/classes/class_node.html>`__\ ) |static| :ref:`🔗<class_TopologyValidator_method_expected_sync_count>`

Returns the minimum expected ``MultiplayerSynchronizer`` count for ``node``. 



Counts standard components present as children: 

- :ref:`SpawnerComponent<class_SpawnerComponent>` -> 1 (extends MultiplayerSynchronizer) 



Does not count user-defined synchronizers; this is a minimum floor only.

`Dictionary <https://docs.godotengine.org/en/stable/classes/class_dictionary.html>`__ **validate_node**\ (\ node\: `Node <https://docs.godotengine.org/en/stable/classes/class_node.html>`__\ ) |static| :ref:`🔗<class_TopologyValidator_method_validate_node>`

Validates the synchronizer topology of ``node``. 



Clears and rebuilds the cache as part of the check. Do not call on a live frame-critical path, this is a diagnostic tool. 



Returns a ``Dictionary``:

::

    {
      "ok":           bool,           # true only when all checks pass
      "errors":       Array[String],  # human-readable descriptions of failures
      "live_count":   int,            # synchronizers found by fresh traversal
      "expected_min": int,            # minimum expected count
    }
